# Automation Test Framework

A comprehensive test automation framework for **Next.js frontend** and **FastAPI backend** applications. Designed for QA/automation testers to create, generate, and run test scripts with minimal manual effort.

## Features

- **Excel-Driven Testing** — Define test cases in Excel, run them automatically
- **Test Script Generator** — Auto-generate Python/Java test scripts from Excel
- **URL Smoke Testing** — Quick smoke test any URL with one command
- **API Testing** — Test FastAPI REST endpoints directly
- **Multi-Browser Support** — Chrome, Firefox, Edge
- **Multi-Language** — Python (pytest + Selenium) and Java (TestNG + Selenium)
- **Keyword-Driven Engine** — 30+ supported actions (click, type, verify, etc.)
- **Batch File Launcher** — Interactive menu, no command-line knowledge needed
- **Auto Screenshots** — Captures screenshots on test failure
- **HTML Reports** — Professional test reports with pass/fail details
- **Excel Results** — Test results exported back to styled Excel files

## Quick Start

### 1. Run Setup

Double-click `setup.bat` or run in terminal:

```
setup.bat
```

This installs all Python dependencies and creates the Excel test case template.

### 2. Configure Your Application

Edit `config\config.ini` with your application URLs:

```ini
[application]
base_url = http://localhost:3000       # Your Next.js frontend URL
api_base_url = http://localhost:8000   # Your FastAPI backend URL
```

### 3. Define Test Cases in Excel

Open `test_data\test_cases.xlsx` and add your test cases using the provided format.

### 4. Run Tests

Double-click `run_tests.bat` to launch the interactive menu:

```
============================================================
   AUTOMATION TEST FRAMEWORK
   Test Script Generator & Runner
============================================================

   1.  Run Tests from Excel File
   2.  Run URL Smoke Test
   3.  Run API Tests (FastAPI)
   4.  Run All Python Tests
   5.  Run Java Tests (TestNG)
   6.  Generate Python Scripts from Excel
   7.  Generate Java Scripts from Excel
   ...
```

## How to Use

### Option 1: Excel-Driven Testing (Recommended)

The easiest approach. Write test steps in Excel, the framework runs them automatically.

**Excel Format:**

| Test_Case_ID | Test_Case_Name | Step_No | Action | Locator_Type | Locator_Value | Test_Data | Expected_Result | Run |
|---|---|---|---|---|---|---|---|---|
| TC001 | Login Test | 1 | navigate | | | http://localhost:3000/login | | Yes |
| TC001 | Login Test | 2 | type | id | username | testuser | | Yes |
| TC001 | Login Test | 3 | type | id | password | testpass123 | | Yes |
| TC001 | Login Test | 4 | click | id | login-btn | | | Yes |
| TC001 | Login Test | 5 | verify_url | | | | /dashboard | Yes |

**Run it:**
```
run_tests.bat → Option 1
```

**Or via command line:**
```
cd python_framework
python -m pytest tests/test_excel_driven.py -v --excel-file=../test_data/test_cases.xlsx
```

### Option 2: URL Smoke Testing

Provide any URL and get instant health checks:

```
run_tests.bat → Option 2 → Enter URL
```

**What it tests:**
- Page loads successfully
- Page title exists
- HTTP status is 200
- Page has content
- No broken images
- Page load time < 10 seconds
- Responsive viewport (Desktop, Tablet, Mobile)
- Screenshot capture

### Option 3: API Testing

Test your FastAPI backend endpoints:

```
run_tests.bat → Option 3 → Enter API URL
```

**What it tests:**
- API reachability
- Health endpoint
- API docs availability (/docs, /redoc)
- CORS headers
- Response time
- Invalid endpoint handling (404)
- JSON content type

### Option 4: Generate Test Scripts from Excel

Convert your Excel test cases into standalone Python/Java test files:

```
run_tests.bat → Option 6 (Python) or 7 (Java) or 8 (Both)
```

This generates complete, runnable test files in `generated_tests/` folder — ready to customize and extend.

## Supported Action Keywords

| Keyword | Description | Locator Required | Test_Data |
|---------|-------------|:---:|---------|
| `navigate` / `go_to_url` | Open a URL | No | URL |
| `click` | Click element | Yes | - |
| `type` / `send_keys` | Type into field | Yes | Text |
| `clear` | Clear input field | Yes | - |
| `select_dropdown` | Select dropdown option | Yes | Option text |
| `verify_text` | Assert element text | Yes | - (use Expected_Result) |
| `verify_title` | Assert page title | No | Expected title |
| `verify_url` | Assert URL contains | No | - (use Expected_Result) |
| `verify_element_present` | Assert element in DOM | Yes | - |
| `verify_element_visible` | Assert element visible | Yes | - |
| `hover` | Mouse hover | Yes | - |
| `double_click` | Double click | Yes | - |
| `right_click` | Right click | Yes | - |
| `wait` / `sleep` | Wait N seconds | No | Seconds |
| `press_key` | Press keyboard key | No | Key name (ENTER, TAB) |
| `scroll_down` / `scroll_up` | Scroll page | No | Pixels |
| `scroll_to_element` | Scroll to element | Yes | - |
| `switch_frame` | Switch to iframe | Yes | - |
| `switch_default` | Back to main content | No | - |
| `accept_alert` / `dismiss_alert` | Handle alerts | No | - |
| `take_screenshot` | Save screenshot | No | Filename |
| `refresh` | Refresh page | No | - |
| `go_back` | Navigate back | No | - |
| `upload_file` | Upload a file | Yes | File path |
| `store_text` | Save element text to variable | Yes | Variable name |
| `execute_js` | Run JavaScript | No | JS code |
| `api_get` | HTTP GET request | No | URL |
| `api_post` | HTTP POST request | No | JSON body |
| `verify_api_status` | Assert API status code | No | - (use Expected_Result) |

## Supported Locator Types

| Type | Description | Example |
|------|-------------|---------|
| `id` | HTML id attribute | `login-btn` |
| `name` | HTML name attribute | `username` |
| `xpath` | XPath expression | `//button[@type="submit"]` |
| `css` | CSS selector | `div.form > input.email` |
| `class` | CSS class name | `btn-primary` |
| `tag` | HTML tag name | `input` |
| `link_text` | Exact link text | `Click Here` |
| `partial_link_text` | Partial link text | `Click` |

## Command Line Usage

### Python Tests

```bash
# Run Excel-driven tests
cd python_framework
python -m pytest tests/test_excel_driven.py -v --excel-file=../test_data/test_cases.xlsx

# Run with specific browser
python -m pytest tests/ -v --browser=firefox

# Run headless
python -m pytest tests/ -v --headless

# Run smoke tests on a URL
python -m pytest tests/test_url_smoke.py -v --test-url=https://example.com

# Run API tests
python -m pytest tests/test_api.py -v --api-url=http://localhost:8000

# Run with markers
python -m pytest tests/ -v -m smoke
python -m pytest tests/ -v -m regression
python -m pytest tests/ -v -m api
```

### Java Tests

```bash
cd java_framework

# Run all tests
mvn clean test

# Run specific test class
mvn test -Dtest=URLSmokeTest

# Pass system properties
mvn test -DtestUrl=http://localhost:3000 -DapiBaseUrl=http://localhost:8000
```

### Script Generation

```bash
# Generate Python scripts
python scripts/generate_test_scripts.py --excel test_data/test_cases.xlsx --language python

# Generate Java scripts
python scripts/generate_test_scripts.py --excel test_data/test_cases.xlsx --language java

# Generate both
python scripts/generate_test_scripts.py --excel test_data/test_cases.xlsx --language both

# Custom output directory
python scripts/generate_test_scripts.py --excel test_data/test_cases.xlsx --language both --output my_tests
```

## Extending the Framework

### Adding Custom Page Objects (Python)

Create a new file in `python_framework/pages/`:

```python
from python_framework.base.base_page import BasePage

class LoginPage(BasePage):
    USERNAME_FIELD = ('id', 'username')
    PASSWORD_FIELD = ('id', 'password')
    LOGIN_BUTTON = ('id', 'login-btn')

    def login(self, username, password):
        self.type_text(*self.USERNAME_FIELD, username)
        self.type_text(*self.PASSWORD_FIELD, password)
        self.click(*self.LOGIN_BUTTON)
```

### Adding Custom Keywords

Edit `python_framework/utils/keyword_engine.py` and add to the `action_map`:

```python
action_map = {
    ...
    'my_custom_action': self._my_custom_action,
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `chromedriver` version mismatch | Framework uses `webdriver-manager` for auto-download |
| Excel file not found | Check the path in `config\config.ini` or pass via `--excel-file` |
| Browser not opening | Ensure the browser is installed and `config.ini` has correct browser name |
| Java tests not running | Ensure JDK 11+ and Maven are installed and in PATH |
| Permission errors | Run terminal as Administrator |
| `ModuleNotFoundError` | Run `setup.bat` to install dependencies |
